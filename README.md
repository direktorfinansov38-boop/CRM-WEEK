## Файлы проекта

```
kimchi_crm/
├── .env              ← СОЗДАЙТЕ САМИ из шаблона (не загружать в git!)
├── .env.example      ← шаблон
├── .htaccess         ← защита файлов + заголовки безопасности
├── config.php        ← читает .env
├── api.php           ← API с CSRF, rate limiting, auth
├── index.php         ← главная (загружает users из БД → вставляет в JS)
├── index.html        ← фронтенд (не изменяйте)
└── install.php       ← установщик (защищён токеном)
```

---

## Шаг 1 — Подготовка файлов

1. Скачайте все файлы проекта
2. Скопируйте `.env.example` → `.env`
3. Откройте `.env` и заполните:
   ```
   DB_HOST=localhost
   DB_NAME=kimchi_crm
   DB_USER=ваш_пользователь_бд
   DB_PASS=ваш_пароль_бд
   INSTALL_TOKEN=придумайте_случайную_строку_например_abc123xyz
   ```

---

## Шаг 2 — Загрузка на Бегит (Beget)

### Вариант А — через FTP/SFTP
1. Подключитесь через FileZilla или другой FTP-клиент
2. Загрузите ВСЕ файлы в папку `public_html/` (или в подпапку, напр. `public_html/crm/`)
3. Убедитесь что `.htaccess` загружен (он скрытый — проверьте настройки FTP-клиента)

### Вариант Б — через Файловый менеджер Бегит
1. Войдите в панель Бегит → Файловый менеджер
2. Перейдите в `public_html`
3. Загрузите все файлы
4. Создайте файл `.env` прямо в менеджере и вставьте содержимое

---

## Шаг 3 — Создание базы данных

В панели Бегит:
1. **MySQL** → Создать базу данных
2. Назовите: `kimchi_crm` (или любое другое — укажите в `.env`)
3. Создайте пользователя БД и запомните пароль
4. Дайте пользователю полные права на эту БД
5. Обновите `.env` с именем пользователя и паролем

---

## Шаг 4 — Запуск установщика

Откройте в браузере:
```
https://ваш-домен.ru/install.php?token=ваш_токен_из_env
```

1. Установщик создаст таблицы автоматически
2. Заполните форму — создайте аккаунт администратора
3. Нажмите "Создать администратора"

---

## Шаг 5 — Защита после установки

**ВАЖНО:** После успешной установки заблокируйте `install.php`.

В файле `.htaccess` раскомментируйте строки:
```apache
<Files "install.php">
    Order allow,deny
    Deny from all
</Files>
```

---

## Шаг 6 — Первый вход

Откройте: `https://ваш-домен.ru/`

- Выберите своё имя в списке (или оставьте поле логина пустым — оно заполнится)
- Введите логин и пароль администратора
- Войдите в систему

## Работа с CRM

### Добавление пользователей
1. Войдите как Администратор
2. Перейдите в раздел **Сотрудники** (иконка людей)
3. Нажмите **+ Добавить сотрудника**
4. Заполните имя, логин, пароль, роль и подразделения
5. Сохраните

### Импорт данных из Excel
1. Войдите в CRM
2. Нажмите кнопку **Импорт** в верхней панели
3. Выберите Excel-файл (плоская таблица)
4. Данные загрузятся автоматически и сохранятся в БД

### Экспорт данных
1. Перейдите в раздел **Экспорт**
2. Выберите подразделение и период
3. Нажмите **Скачать Excel**

## Требования сервера

- PHP 8.0+
- MySQL 5.7+ или MariaDB 10.4+
- mod_rewrite (Apache) — для .htaccess
- Расширения PHP: `pdo_mysql`, `json`, `session`

На Бегит всё это есть по умолчанию.

## Безопасность — что делать точно

- [ ] Никогда не добавляйте `.env` в git-репозиторий
